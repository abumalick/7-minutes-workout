## Overview

A single-page "7 Minute Workout" timer built with React 19, TanStack Router (file-based), Vite, and Tailwind CSS v4. The entire app is one route: a timed exercise sequence with start/pause/next/previous controls and audio cues.

## Package manager

This project standardizes on **bun** (`bun.lock`, README uses `bun`/`bunx`). Use bun, not pnpm/npm/yarn, here.

## Commands

- `bun run dev` / `bun run start` — dev server on port 3000
- `bun run build` — `vite build` then `tsc` (type-check as part of build)
- `bun run serve` — preview the production build
- `bun run test` — run Vitest once (jsdom env, globals enabled)
- `bun run test -- <path>` — run a single test file (e.g. `bun run test -- src/hooks/useInterval.test.ts`)
- `bun run lint` — Biome check+fix (`lint:js`) then `tsc --noEmit` (`lint:ts`)

Add shadcn/ui components with `bunx shadcn@latest add <component>` (style: new-york, base color: zinc, into `@/components/ui`).

## Architecture

- **Routing**: TanStack Router file-based, in `src/routes/`. `routeTree.gen.ts` is auto-generated by the Vite plugin (`autoCodeSplitting: true`) — never edit it; it's also excluded from Biome. `src/routes/__root.tsx` is the layout (renders `<Outlet />` + devtools).
- **`-components/` convention**: Directories/files prefixed with `-` inside `src/routes/` are excluded from routing (they're plain components, not routes). Shared route components live in `src/routes/-components/`.
- **All app logic lives in `src/routes/index.tsx`** (the `/` route). It owns `WORKOUT_SEQUENCE` (static array of `{ label, duration }`) and all state: `currentIndex`, `timeLeft`, `isRunning`. `WorkoutPanel` and `Controls` are presentational — they receive values and callbacks as props.
- **Timer**: driven by `src/hooks/useInterval.ts` (Dan Abramov's declarative `setInterval` hook). Passing `delay: null` pauses it; the route passes `isRunning ? 1000 : null`. Countdown, auto-advance to the next exercise, and end-of-sequence stop all happen inside the interval callback.
- **Audio cues**: three `Audio` objects (`start`/`tick`/`success`) memoized in `index.tsx`, imported as mp3 assets from `src/assets/sounds/`. `tick` plays for the last 5 seconds of a step; `start`/`success` play on entering/finishing non-`Rest` steps. `"Rest"` is a magic label used throughout to distinguish rest from exercise.

## Conventions

- **Biome** for lint + format: single quotes, no semicolons, space indentation, organize-imports on. Recommended rules enabled.
- **TypeScript** is strict, with `noUnusedLocals`/`noUnusedParameters` and `verbatimModuleSyntax`. `@/*` aliases `./src/*`.
- Component prop types are inlined as object type annotations on the function signature (no separate `interface`/`type` declarations) — match this style.
